# Task Workflow

This project uses a `task.md` file at the root to define work items.

## Structure

Each `task.md` contains:
- A clear description of what needs to be done
- Acceptance criteria or success conditions
- Any constraints or context

## Execution

1. Read `task.md` fully before starting.
2. Break the work into clear, sequential steps.
3. Verify each step works before moving to the next.
4. Update `task.md` with progress as you go (mark completed items).
5. When done, confirm all acceptance criteria are met and summarize what was done.
